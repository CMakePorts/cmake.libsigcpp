Source Packages

The source packages for libsigc++ are available on the sourceforge download area and on the gnome ftp site.

Binary Packages

Whenever possible, you should use the official binary packages approved by the supplier of your operating system, such as your Linux distribution. For instance, Debian Linux and Fedora Linux have official libsigc++ packages.

Requirements

 pkg-config should be used to build software that depends on libsigc++.

libsigc++ is built and tested for a standards-compliant C++ compiler. Compilers that don't fully implement the the template sections of the standard are not likely to work properly. But we welcome any patches that might work around compiler problems.

Required from the C++ compiler:

  • Namespaces.
  • Partial specialization of class and function templates.
  • Member function templates and explicit instantiation.
  • Template copy constructors and operators.
  • Treat void return as normal return.

All version of gcc >= 3.2 are currently known to compile libsigc++-2.0. Beginning with libsigc++-2.0.1 the library also works with the Apple gcc port. Also cygwin (gcc >= 3.2) and mingw32 should be able to compile libsigc++-2.0.x. Project files for the Microsoft .NET 2003 C++ compiler are integrated into the tarball. Support for SUN FORTE C++ starts with libsigc++-2.0.4. libsigc++-2.0.1 was the first tarball that has been reported to compile with the Intel C++ compiler.

A larger number of compilers have been tested and found capable for libsigc++-1.2. All versions of gcc (including cygwin) passed enough tests to compile. HP-UX aCC used to lack member function copy constructors. A port to Microsoft Visual C++ has been integerated into the distribution. The primary development platform is gcc.

Although not required, m4 is useful to construct templates not provided in the standard package.
